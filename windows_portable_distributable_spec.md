# Windows Portable AI App — Distributable Specification
Version: 1.1

For application architecture and development requirements, read `windows_portable_app_spec.md`.

---

# Purpose

This document defines how to package a Windows Portable AI App into a distributable ZIP that runs on any Windows 11 machine without installation.

The goal is to ensure every packaged application:

- Runs on Windows 11 without Python, Node.js, or admin rights
- Launches via double-click from any user-accessible folder
- Is distributable as a single ZIP file
- Requires zero setup from the end user

The AI coding agent MUST follow this specification unless explicitly overridden.

---

# Engineer Starting Point

The distribution engineer receives a completed `AppName/` application folder from the developer.

**Steps before starting the AI coding session:**

1. Create a new parent folder (name it anything — this becomes the working repo)
2. Copy the `AppName/` application folder into it as a subfolder
3. Copy `windows_portable_distributable_spec.md` and `CLAUDE.md` into the parent folder root
4. Open the AI coding session in the parent folder

The AI agent will build the `dist/` folder alongside `AppName/` and produce the distributable.

The parent folder will end up with exactly two sub-folders:

```text
parent-folder/
├── AppName/       ← application source (provided by developer — do not modify)
└── dist/          ← all distribution tooling and output (built by AI agent)
```

Replace `AppName` throughout with the actual project folder name.

---

# Core Distribution Philosophy

The recipient of the application should:

1. Receive a ZIP file
2. Extract the ZIP file
3. Double-click `Run_App.bat`
4. Immediately use the application

The recipient should NEVER need to:

- Install Python
- Install Node.js
- Open a terminal
- Run CLI commands
- Configure environment variables
- Install Docker
- Use administrator privileges

All required runtimes and dependencies MUST be bundled at package time.

# Required Project Structure

The repo MUST have exactly two sub-folders. Nothing else lives at the root except `CLAUDE.md`.

```text
repo/
├── AppName/                  Core application source — named after the project
│   ├── backend/              Python FastAPI app — business logic, API routes, database access
│   ├── frontend/             React/Vite source — compiled at build time, never shipped as source
│   │   ├── package.json      Pinned frontend dependencies (no ^ or ~ version ranges)
│   │   ├── vite.config.js    Vite build configuration
│   │   └── tailwind.config.js
│   ├── assets/               Static files that ship with the app (icons, images, fonts)
│   ├── data/                 Seed data and initial schema scripts used at first run
│   ├── launcher/             Main entry point — starts FastAPI, opens PyWebView, manages shutdown
│   ├── requirements.txt      Pinned Python dependencies (exact == versions only)
│   └── README.md             Project documentation
└── dist/                     All distribution tooling and output
    ├── build.bat             Developer script — installs deps and compiles the React frontend
    ├── package.bat           Developer script — runs build.bat + PyInstaller, produces dist/AppName/
    ├── Run_App.bat           End-user launcher — staged here, copied into AppName/ at package time
    ├── packaging/
    │   └── app.spec          PyInstaller config — controls what gets bundled
    ├── AppName/              Built output folder (generated by package.bat — this is what gets zipped)
    └── AppName.zip           Final distributable ZIP
```

Key distinctions:

- `dist/` is the **single boundary** between developer and end user. All packaging scripts, configuration, and output live here.
- `frontend/` is **build-time only**. The compiled static assets are bundled — the source is never shipped.
- `data/` holds **seed/schema data** for first run. It is NOT for runtime user data — that goes under `%LOCALAPPDATA%/AppName`.
- `CLAUDE.md` stays at the repo root for Claude Code auto-loading.

# PyInstaller Configuration Requirements

## console=False

The PyInstaller `.spec` MUST set `console=False` to suppress the terminal window.

```python
exe = EXE(
    ...
    console=False,
    ...
)
```

Without this, a terminal window appears behind the application on launch.

---

## stdout/stderr Encoding in Windowed Mode

Setting `console=False` causes Python's stdout and stderr to inherit the Windows charmap (cp1252) encoding. Any `print()` call containing non-ASCII characters — article titles, filenames, exception messages from external data — will raise `UnicodeEncodeError` and crash background tasks silently.

The launcher MUST redirect stdout and stderr to UTF-8 at the very top of `launcher/main.py`, before any other imports:

```python
import io as _io, os as _os, sys as _sys

if getattr(_sys, "frozen", False):
    _nul = open(_os.devnull, "w", encoding="utf-8", errors="replace")
    _sys.stdout = _nul
    _sys.stderr = _nul
else:
    for _attr in ("stdout", "stderr"):
        _s = getattr(_sys, _attr, None)
        if _s and hasattr(_s, "reconfigure"):
            try: _s.reconfigure(encoding="utf-8", errors="replace")
            except Exception: pass
```

Any backend module that prints user-supplied or externally-sourced content (titles, URLs, API responses, exception messages) MUST use an encoding-safe wrapper instead of bare `print()`:

```python
def _print(*args, **kwargs):
    try:
        print(*args, **kwargs)
    except (UnicodeEncodeError, AttributeError):
        safe = " ".join(
            a.encode("ascii", errors="replace").decode("ascii")
            if isinstance(a, str) else str(a) for a in args
        )
        try:
            print(safe, **{k: v for k, v in kwargs.items() if k != "end"})
        except Exception:
            pass
```

## MSVC Runtime DLL Requirements

PyInstaller does not always bundle the Microsoft Visual C++ runtime DLLs automatically. On stripped-down or corporate-hardened Windows machines these DLLs may be absent, causing the application to crash immediately — before Python starts — with no useful error message.

The `.spec` file MUST explicitly include the MSVC runtime DLLs as `binaries`:

```python
import os, sys

msvc_dlls = []
for dll in ["MSVCP140.dll", "VCRUNTIME140.dll", "VCRUNTIME140_1.dll"]:
    for search_dir in [sys.base_prefix, r"C:\Windows\System32"]:
        candidate = os.path.join(search_dir, dll)
        if os.path.exists(candidate):
            msvc_dlls.append((candidate, "."))
            break

a = Analysis(
    ...
    binaries=msvc_dlls,
    ...
)
```

## app.spec project_root Pattern

The spec file lives at `dist/packaging/app.spec`. It MUST calculate the project root by going up two levels to the repo root, then into the `AppName/` sub-folder. Replace `AppName` with the actual project folder name.

```python
_repo_root = os.path.abspath(
    os.path.join(os.path.dirname(os.path.abspath(SPEC)), "..", "..")
)
project_root = os.path.join(_repo_root, "AppName")
```

Full example spec:

```python
# -*- mode: python ; coding: utf-8 -*-
import os, sys

block_cipher = None

# SPEC lives at dist/packaging/app.spec — go up two levels to repo root, then into AppName/
_repo_root = os.path.abspath(os.path.join(os.path.dirname(os.path.abspath(SPEC)), "..", ".."))
project_root = os.path.join(_repo_root, "AppName")

# Locate MSVC runtime DLLs
msvc_dlls = []
for dll in ["MSVCP140.dll", "VCRUNTIME140.dll", "VCRUNTIME140_1.dll"]:
    for search_dir in [sys.base_prefix, r"C:\Windows\System32"]:
        candidate = os.path.join(search_dir, dll)
        if os.path.exists(candidate):
            msvc_dlls.append((candidate, "."))
            break

a = Analysis(
    [os.path.join(project_root, "launcher", "main.py")],
    pathex=[project_root],
    binaries=msvc_dlls,
    datas=[
        (os.path.join(project_root, "frontend", "dist"), "frontend/dist"),
        (os.path.join(project_root, "data"),             "data"),
    ],
    hiddenimports=[
        "uvicorn.logging",
        "uvicorn.loops",
        "uvicorn.loops.auto",
        "uvicorn.protocols",
        "uvicorn.protocols.http",
        "uvicorn.protocols.http.auto",
        "uvicorn.protocols.websockets",
        "uvicorn.protocols.websockets.auto",
        "uvicorn.lifespan",
        "uvicorn.lifespan.on",
        "uvicorn.lifespan.off",
        "uvicorn.main",
        "sqlalchemy.dialects.sqlite",
        "backend.main",
    ],
    excludes=[
        "websockets",
        "torch", "torchvision", "torchaudio",
        "cv2", "scipy", "sklearn",
        "matplotlib", "matplotlib.backends",
        "PIL", "Pillow",
        "pandas", "pyarrow",
        "transformers", "tokenizers",
        "grpc", "grpcio",
        "opentelemetry",
        "pytest", "py",
        "tensorflow", "keras",
    ],
    cipher=block_cipher,
    noarchive=False,
)

pyz = PYZ(a.pure, a.zipped_data, cipher=block_cipher)

exe = EXE(
    pyz,
    a.scripts,
    [],
    exclude_binaries=True,
    name="AppName",
    debug=False,
    strip=False,
    upx=True,
    console=False,
    icon=os.path.join(project_root, "assets", "icon.ico")
         if os.path.exists(os.path.join(project_root, "assets", "icon.ico"))
         else None,
)

coll = COLLECT(
    exe,
    a.binaries,
    a.zipfiles,
    a.datas,
    strip=False,
    upx=True,
    upx_exclude=[],
    name="AppName",
)
```

Add any project-specific `hiddenimports` as needed. The `excludes` list prevents large ML/data-science packages on the developer's machine from inflating the bundle.

# Build Script Requirements

All packaging scripts MUST live inside `dist/`. This keeps the project root clean and makes `dist/` the single location for everything distribution-related.

Replace `AppName` throughout with the actual project folder name.

---

## dist/build.bat

Installs Python and frontend dependencies, then compiles the React frontend.

- MUST use `%~dp0..\AppName\` to reference the source sub-folder
- Steps in order:
  1. `pip install -r "%~dp0..\AppName\requirements.txt"`
  2. `pushd "%~dp0..\AppName\frontend" && npm install`
  3. `npm run build` (produces `AppName/frontend/dist/`)

```bat
@echo off
setlocal
pip install -r "%~dp0..\AppName\requirements.txt"
if %errorlevel% neq 0 exit /b 1
pushd "%~dp0..\AppName\frontend"
npm install
if %errorlevel% neq 0 ( popd & exit /b 1 )
npm run build
if %errorlevel% neq 0 ( popd & exit /b 1 )
popd
```

---

## dist/package.bat

Calls `build.bat`, runs PyInstaller, stages `Run_App.bat`, and copies output to `dist/AppName/`.

- MUST redirect PyInstaller `--workpath` and `--distpath` to `%TEMP%` to avoid OneDrive or antivirus file locks
- Copies `dist/Run_App.bat` into the PyInstaller output before robocopy
- Uses robocopy to move output from `%TEMP%` to `dist/AppName/`
- MUST use `if %errorlevel% geq 8` for robocopy — exit code 1 means success (files copied), NOT failure

```bat
@echo off
setlocal
call "%~dp0build.bat"
if %errorlevel% neq 0 exit /b 1

set DIST_TEMP=%TEMP%\AppName-dist
set WORK_TEMP=%TEMP%\AppName-pyinstaller

python -m PyInstaller "%~dp0packaging\app.spec" --distpath "%DIST_TEMP%" --workpath "%WORK_TEMP%" --clean -y
if %errorlevel% neq 0 exit /b 1

copy /Y "%~dp0Run_App.bat" "%DIST_TEMP%\AppName\Run_App.bat"

if exist "%~dp0AppName" rmdir /s /q "%~dp0AppName"
robocopy "%DIST_TEMP%\AppName" "%~dp0AppName" /E /NFL /NDL /NJH /NJS /NC /NS /NP
if %errorlevel% geq 8 exit /b 1
exit /b 0
```

---

## dist/Run_App.bat

End-user launcher. Staged in `dist/` and copied into `dist/AppName/` by `package.bat` at build time.

- MUST check for WebView2 before launching (see WebView2 Preflight Check below)
- Launches `AppName.exe` relative to its own location using `start "" "%~dp0AppName.exe"`
- Paths are relative to where the file runs **after extraction** — not to where it lives during development

```bat
@echo off
setlocal

reg query "HKLM\SOFTWARE\WOW6432Node\Microsoft\EdgeUpdate\Clients\{F3017226-FE2A-4295-8BDF-00C3A9A7E4C5}" >nul 2>&1
if %errorlevel% neq 0 (
    echo.
    echo  This application requires the Microsoft Edge WebView2 Runtime.
    echo  It does not appear to be installed on this machine.
    echo.
    echo  Please download and install it from:
    echo  https://developer.microsoft.com/en-us/microsoft-edge/webview2/
    echo.
    pause
    exit /b 1
)

start "" "%~dp0AppName.exe"
```

---

## WebView2 Preflight Check

PyWebView depends on the Microsoft Edge WebView2 runtime. On Windows 11 it is pre-installed. On Windows 10 it may be absent.

`Run_App.bat` MUST check for WebView2 before launching. If absent, it MUST display a clear, user-friendly error message with the download URL. It MUST NOT silently fail.

The registry key to check:

```
HKLM\SOFTWARE\WOW6432Node\Microsoft\EdgeUpdate\Clients\{F3017226-FE2A-4295-8BDF-00C3A9A7E4C5}
```

# Required Distribution Output

Running `dist/package.bat` MUST produce:

```text
dist/AppName/       ← runnable on any Windows 11 machine without modification
dist/AppName.zip    ← zip this folder and share it
```

`dist/AppName/` contains:

```text
AppName/
├── Run_App.bat         ← end-user launcher (double-click to start)
├── AppName.exe         ← PyInstaller executable (bootstraps Python runtime)
└── _internal/          ← Python runtime + all dependencies
    ├── python3xx.dll   ← bundled Python runtime
    ├── (pip packages)  ← all Python libraries
    ├── frontend/dist/  ← compiled React build (HTML, CSS, JS)
    ├── data/           ← seed data and schema
    ├── assets/         ← icons, images, fonts
    └── (DLLs)          ← SQLite, MSVC runtime, native libraries
```

The output folder MUST be runnable on another Windows machine without modification.

# Packaging Workflow Summary

```text
Developer machine (has Python + Node.js)
  └── dist\package.bat
        ├── Calls dist\build.bat
        │     ├── pip install -r ..\AppName\requirements.txt
        │     ├── npm install (in ..\AppName\frontend)
        │     └── npm run build  →  AppName/frontend/dist/
        ├── PyInstaller (--distpath %TEMP%, --workpath %TEMP%)
        │     └── Bundles: Python runtime + packages + frontend/dist + data + DLLs
        ├── Copies dist\Run_App.bat into %TEMP%\AppName-dist\AppName\
        └── Robocopy %TEMP%\AppName-dist\AppName → dist\AppName\

Output:
  dist\AppName\        ← runnable folder
  dist\AppName.zip     ← share this
```
